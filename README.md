DESCRIPTION : 
This Java application is a REST API client designed to fetch weather information from a public web service and display the retrieved data in a structured and user-friendly format. The main purpose of this project is to demonstrate how Java applications can communicate with external APIs through HTTP requests and process JSON responses. The program uses the OpenWeather API as the data source and retrieves real-time weather information for a specified city. In this implementation, the selected city is Pune. The application showcases the practical use of networking concepts, API integration, and JSON data parsing in Java.

The program begins by importing required Java classes such as BufferedReader, InputStreamReader, HttpURLConnection, and URL. These classes are necessary for establishing a network connection, sending requests, and reading responses from the API server. Additionally, the application imports the JSONObject class from the org.json library. This library helps convert raw JSON data into Java objects so that specific information can be extracted easily.

The application defines a class named WeatherAPIClient, which contains the main method responsible for executing the program. Inside the class, an API key variable is declared as a constant. The API key is obtained from the OpenWeather platform and acts as an authentication token that allows the application to access the weather service. Without a valid API key, the server will reject the request.

Within the main() method, a city name is specified. The program dynamically creates a request URL by combining the base API endpoint, city name, API key, and temperature unit parameters. The URL generated by the application serves as the endpoint for requesting weather information from the server.

The program then creates a URL object and opens an HttpURLConnection to establish communication with the API server. The request method is set to GET, which indicates that the client wants to retrieve information rather than send or modify data. After sending the request, the application checks the HTTP response code. A response code of 200 indicates that the request was successful.

Once a successful connection is established, the application reads the response received from the server. The BufferedReader class reads the incoming data line by line and stores it in a StringBuilder object. Since the API returns weather details in JSON format, the response is converted into a JSONObject for easier processing.

After parsing the JSON data, the program extracts important weather information such as city name, temperature, humidity, and weather description. These values are accessed through JSON methods and stored in variables. The extracted data is then displayed in a structured weather report format using System.out.println() statements. This organized presentation improves readability and provides users with clear information.

The application also includes exception handling using a try-catch block. Exception handling prevents the program from terminating unexpectedly when errors occur, such as invalid API keys, internet connectivity issues, or incorrect city names. If an exception is detected, Java displays an error message to help identify the problem.

Overall, this project demonstrates the implementation of a Java REST API client capable of sending HTTP requests, consuming public APIs, parsing JSON responses, and displaying formatted results. It provides hands-on experience with API integration and modern web communication techniques in Java.
